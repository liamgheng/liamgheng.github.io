避免短期焦虑的方法是看向未来。

很多软件工程师会焦虑之后的出路，担心自己的竞争力，我也会担忧，尤其是最近半年没怎么贡献代码，会尤其担忧未来竞争力。

破解焦虑的方法之一是想清楚市场需要怎样的工程师，我会选择哪些工程师做我的同事。

## 基础扎实

- debug 经验丰富，能够快速处理线上故障。
- 计算机网络基础扎实，二层，三层，四层，七层协议信手拈来，容器网络，Paas 网络，混合云和公共云网络架构都有所涉猎。
- 操作系统扎实，了解操作系统内部实现，了解死锁，虚拟内存，I/O相关。
- 编程语言扎实，了解多种编程范式并能灵活组合写出优雅的代码，了解编程语言背后实现最好。
- 算法扎实，了解常用算法及其在业界的使用，知道根据业务场景选择合适的算法。
- 良好的编码风格：合理的注释，设计文档，Commit Msessage，Unit Test。

## 视野开阔

- 了解行业内的优秀企业是如何做的，并能搞清楚当前团队的差距在哪里。
- 了解其他团队的各种工程实践：数据库，微服务，监控，云原生等等等等。
- 了解技术演变过程，知道应该在哪个方向投入精力，哪个方向是短期形态不值得投入精力。

## 沟通，表达，落地能力

- 如何清晰表达，包括语音和文字场景。
- 如何与不通角色沟通。
- 如何激励。
- 如何把事情落地，需要项目管理能力，协调不同团队利害，团结把事情做成的能力。
- 给团队带来正向的变化，比如更好的 Code Review，确保做高优先级事情来提高整体产出。
